Taking Notes in Tailwind

Creating and Styling Lists

  • List items must have enough margin on the x-axis for the bullet point to show up. I often use mx-6.
  • List items must be held within a <ul> element. This element is where we can specify the margin and list-style classes.
  • Writing a simple <p> element and specifying class as list-disc won't create a list.

Writing Literal Code

  • Writing literal code has some similarities, and some differences when styling with Tailwind. For including symbols, we can still use the "&lt;" and "&gt;" for the brackets, but the styling is not done using a <code> element, it is done with custom Tailwind class styling. See this .html file for this style.

2.1) Intro

  • In the first introductory week of the Bootcamp we’ve learned the basics of creating HTML pages and styling them with Tailwind. We also learned how to publish a static HTML page on Netlify.The term 'static' here is in contrast with 'dynamic'. Dynamic pages are typically generated with the use of a database, and get their final state different depending on some data that changes.

2.2) Introducing Astro

  • Astro is a "modern static site builder". it makes things very easy to work with HTML and Tailwind, and to have things with a proper structure right from the start.
  • And by implementing it you start working with your first development tool, which is something you’ll get used to all the time. As developers, one of our tasks is to always try new tools.
  • Astro is a tiny abstraction over the HTML files we used in the last week when we wrote HTML in a .html file.

    • See the Mozilla definition of "abstraction" here.

    Stackify article and example of abstraction here.

    "I’m a coffee addict. So, when I wake up in the morning, I go into my kitchen, switch on the coffee machine and make coffee. Sounds familiar? Making coffee with a coffee machine is a good example of abstraction. You need to know how to use your coffee machine to make coffee. You need to provide water and coffee beans, switch it on and select the kind of coffee you want to get. The thing you don’t need to know is how the coffee machine is working internally to brew a fresh cup of delicious coffee. You don’t need to know the ideal temperature of the water or the amount of ground coffee you need to use. Someone else worried about that and created a coffee machine that now acts as an abstraction and hides all these details. You just interact with a simple interface that doesn’t require any knowledge about the internal implementation. You can use the same concept in object-oriented programming languages like Java."

  • What we’ll do in Astro is, we’ll just write the same HTML but we’ll create .astro files intead of .html pages.
  • That’s it (to start with). Just a file extension change.
  • In this project we’ll write multiple HTML pages in Astro. But before we can install Astro, we must install Node.js because we need to use the npm command line tool.

2.4) Set up and Astro Project and create a Git repository

Node.js

  • Now that you have Node.js installed, you have access, among others, to the node and npm commands.
  • node lets you run Javascript files.
  • You can type node file.js and the JavaScript contained in the file.js file will be executed.
  • npm is a utility program that is used to mainly do 3 things:
    • Initialize a project using npm init <...>
    • Download and install Javascript packages, and add them to your project using npm install <...>
    • Run a project using npm run <...>
  • You’ll use this command all the time. Especially you’ll use npm run dev a lot, which is the command we use to run tools like Astro or Next.js locally, for development.
