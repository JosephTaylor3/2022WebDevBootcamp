1.6.) HTML Body Tags

Inline and Block Elements

  • Tags can be separated into INLINE and BLOCK elements.
    • Block elements (<p>, <div>, heading elements, lists and list items, and others) when positioned in the page do not allow other elements to be visualized next to them. Inline elements instead can sit next to other inline elements. Another difference is that inline elements can be contained in block elements. The reverse is not true.
      • Example: Inside a <p> tag, we can add any inline element we like, like <span>.
      • We cannot add other block elements into a <p> or <span> tag.
      • We cannot nest a <p> element into another one, as <p> is a block tag.

Links

Links are defined using the <a> tag. The link destination is set via its href attribute.

Link tags can include any other element inside of it, except for another <a> tag.

Attributes

  • Attributes have the key="value" syntax.
  • And some attributes are boolean, meaning you only need the key.
  • The class and id attributes are two of the most common you will find used.
    • The difference between the two is that an id is unique in the context of a web page; it cannot be duplicated. Classes, on the other hand, can appear multiple times on multiple elements. Plus, an id is just one value. class can hold multiple values, separated by a space:

      • <p class="class-1 class-2 another-class" > A paragraph of text </p>

Images

The HTML standard for image elements requires an alt attribute to be present, to describe the image. This is used by screen readers and also by search engine bots:

<img src="dog.png" alt="A picture of a dog" />

Container Tags

HTML provides a set of container tags. Those are block tags that can contain other tags, and are used to organize the information on a page.

The tags are:

  • <article> identifies a thing that can be independent from other things in a page.
  • <section> represents a section of a long article. Each section has a heading tag (h1-h6), then the section content.
  • <div> is the generic container element. It's the one we use when we don't have another dedicated container tag like <article>.
  • <nav> is used to create the markup that defines the page navigation or site menu. Into this we typically add an ul or ol list of links to other sections of the website or page.
  • <aside> is used to add a piece of content that is related to the main content; a quote or sidebar perhaps. Using <aside> is a signal that the things it contains are not part of the regular flow of the section it lives into.
  • <header> represents a part of the page that is the introduction.
  • <main> represents the main part of a page.
  • <footer> is used to determine the footer of an article, or the footer of the page.

1.7.) HTML Head Tags

The <head> tag always appears right after the opening html tag and before the <body> tag. The <head> tag contains special tags that define the document properties. We never use attributes on this tag. And we don't write content in it.It's just a container for other tags.

The different head tags are:

  • <title> determines the page title. The title is displayed in the browser, and it's especially important as it's one of the key factors for Search Engine Optimization (SEO).
  • <script> is used to add JavaScript into the page, either inline between <script> tags, or via a src attribute, where <script src="filepath.js"></script>.
  • <link> is mainly used to link an external CSS file. It's a self-closing tag. For example, <link href="file.css" rel="stylesheet" />. We can also link to resources other than stylesheets, such as links to the HTML & CSS icon that shows in the browser tab for this webpage (see the header of this .html file)
  • <style> can be used to add CSS styles into the document, rather than loading an external stylesheet.
  • <meta> tags performa a variety of tasks. The most common is the description tag, such as <meta name="description" content="A nice page" />. This should be used by Google to generate the page description in its result pages, although these days its algorithm is a bit.. fancier. The charset meta tag, <meta charset="utf-8" /> is used to set the page character encoding, which in the western world is typically set to utf-8.

1.8) Using Browser DevTools

One of the important tools of the DevTools in each browser, aside from the Elements tab, is the Console. The Console serves mainly two purposes: executing JavaScript code manually, and seeing any JavaScript error reporting.

The devtools embed a very useful device emulator which you can use to visualize your page in every device size you want.

The Network tab provides information on the network requests made by the browser, and the responses that it got. It’s very useful.

1.9) Styling HTML Pages (with CSS)

  • CSS is a standard and it stands for Cascading Style Sheets.
  • CSS is a VAST topic, and one where you can spend lots of time mastering. But, we’re not going to look at it much in the Bootcamp. We do use CSS, but we’ll work on a different abstraction level. Much like we’ll do next with JavaScript. Instead of building interfaces in the browser using JavaScript directly, we’ll use React, which is an abstraction that makes our lives easier. We’ll do the same with CSS by using Tailwind. Tailwind is a way to apply CSS by using HTML classes.
  • Instead of manually writing an external or in-header style sheet, we can use the class attribute on HTML tags. For example, <p class="text-red-500">test</p>
  • Note that we’ll need to load Tailwind in our project, either by loading the CSS file directly, which is fine in testing, or using the Tailwind-specific tooling that every web app framework like Next.js provides.
  • Each class is tied to just one CSS instruction, usually. What we like about that is we write the style in the HTML directly.
  • The HTML can get kind of clunky, particularly when you are assigning multiple classes, but it gives us strong customizability, directly inside the HTML.
  • And one of the main benefits is the maintainability. The HTML directly describes the presentation, which is great with components-based UI layouts design, which is what we’ll focus on for the entire bootcamp.

But it will not pick up on new classes until you run npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch )make sure your file paths are specific to how you have structured and named your files), where the "--watch" is VERY important, because it will keep you from having to go back and run your build command every single time you add a new class to your html file, and it will "Watch" for new classes and update the output style sheet as needed based on the classes in the file.

Miscellaneous Notes

Taking Notes in HTML and Producing Literal Code Output

Creating nested (cascading) lists in HTML:

Inside of an unordered list, <ul>, you will naturally have list items, <li>. But you can then add another unordered list inside of any of your list items, like so:

<ul>

  • <li> list item one </li>
  • <li> list item two with subitems:

      <ul> ul 2:

    • <li> Subitem 1
    • <li> Subitem 2

      • </ul>

    • </li>

</ul>

To display literal HTML code in the webpage:

Writing tags:
  • First, type "&amp;" instead of just "&" at the beginning of entities and symbol code. (note that this line uses "&amp;" before each "&amp;" in the actual code so that the second "amp;" will actually show up literally :D )
  • We want to type: "&lt;" for the opening < tag,
  • And we want to type: "&gt;" for the closing > tag.
  • And then we want to wrap it all in < code > tags so that we get a code-style formatting for our output, like so: <body>
Writing Tags - Examples:
  • So, if we want to write a span tag that shows up literally, we can write "&lt; span &gt;". (Note again that we used "&amp;" before the "lt;" and "gt;" so this would show literally. Without the extra preceding "amp;" before the lt; and gt;, and wrapped in our < code > tags, we'd get our desired output of: <span>)
  • And another example: "&lt;a href="https://flaviocopes.com" &gt; Click Here &lt;/a&gt;" (literal) produces:
    <a href="https://flaviocopes.com" > Click Here </a>      (desired output - preceding "amp;"s removed).

Creating Spaces in HTML display:
  • Type: "&nbsp;" wherever you need an extra space
  • So, "&lt;a href="#"&gt; test &lt;/a&gt;" produces:    <a href="#"> test </a>
